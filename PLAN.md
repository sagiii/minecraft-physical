# PLAN.md — ロードマップ

## ビジョン

MinecraftワールドとIoT/物理デバイスをMQTTで対等に接続する**汎用ブリッジツールボックス**。

- ゲーム内オブジェクト（ブロック、エンティティ、環境）とリアルデバイス（センサー、アクチュエータ、ディスプレイ）が同じMQTTブローカー上に並ぶ
- Minecraftを「デジタルツインのキャンバス」として活用できる
- 小学生低学年でも遊びながらIoTの概念（信号・センサー・アクチュエータ・ネットワーク）を体験できる教材としても機能する
- ローカル環境からリモートブローカーに拡張すると、世界中のデバイス・ワールドが相互接続できる

## フェーズ一覧

### Phase 0 — 基盤 ✅ 完了

- Fabric MOD + MQTTブリッジ基盤
- チャンネルIN/OUTブロック (boolean信号、ID 1〜99)
- MicroPythonライブラリ (`minecraft_mqtt.py`) + UIFlowカスタムブロック
- MQTT監視ツール (`mqtt-monitor.html`)
- トピック命名を `mp/bridge/...` 体系に統一

### Phase 1 — カメラストリーミング 🚧 進行中

**目標:** Minecraftワールドの映像を実デバイスのディスプレイに表示する

- [ ] Fabric カメラエンティティ実装
  - オフスクリーンレンダリング (Fabric Rendering API)
  - JPEG圧縮 + MQTTパブリッシュ
  - `ctrl` トピック購読 (解像度・FOV・FPS・pan/tilt 制御)
- [ ] M5StickC Plus デモ (240×135px、フォトフレーム/スマートバッジ)
- [ ] `mqtt-monitor.html` にカメラプレビュー追加

### Phase 2 — 環境・モブセンサー 📋 計画中

**目標:** Minecraft内の状態変化を実世界に通知する

- [ ] モブセンサー: 近傍エンティティ情報を `mp/bridge/m/mob/{id}` で配信
- [ ] 環境センサー: 天候・時刻・バイオームを `mp/bridge/m/env/{id}` で配信
- [ ] M5Stack向けサンプル (モブ接近でバイブレーション、天気同期LEDなど)

### Phase 3 — 双方向リッチデータ 📋 計画中

**目標:** boolean以外のデータ型をサポートする

- [ ] アナログ値: 実世界センサー値をMinecraft内変数/天候/パーティクルに反映
- [ ] テキスト: チャット連携
- [ ] 実世界カメラ → Minecraftマップアート投影 (PC側Pythonスクリプト経由)

### Phase 4 — スケールアップ 💡 将来構想

- リモートMQTTブローカー対応 (HiveMQ Cloud等)
- マルチインスタンス対応 (`mp/bridge/{instance}/...`)
- Web UI の強化 (カメラ表示、センサーグラフ、テラリウムビュー)
- M5Dial によるカメラコントローラー (ロータリーエンコーダー → pan/zoom)
